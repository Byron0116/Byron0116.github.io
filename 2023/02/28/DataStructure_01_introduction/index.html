数据结构的概念

基本概念

  • 数据(data)
    数据是 信息的载体，是描述客观事物属性的数、字符及所有 能输入到计算机中并被计算机程序识别和处理的符号的集合。
  • 数据元素(data element)
    数据元素是数据的基本单位，通常作为一个整体进行考虑和处理。
  • 数据项(data item)
    一个数据元素可由若干数据项组成，数据项是构成数据元素的不可分割的最小单位。
    数据项又可分为：
    1. 初等项
      在数据处理时不能再分割的最小单位。
    2. 组合项
      由初等项组合而成。

例如，对于学籍管理文件来说，每一个学生记录就是数据元素，而学生的性别、籍贯、成绩等就是数据项。
而性别、籍贯为初等项，学生成绩为组合项，可划分为数学成绩、英语成绩等初等项。

  • 结构
    在数据处理中涉及的数据元素之间都不会是孤立的，数据元素之间的 关系 称为结构。

    例如，在n个网站之间建立通信网络，要求网络中任一网站出现故障，整个网络仍然保持畅通，这样所有网站之间形成网状关 系。

  • 数据类型
    类型是一组值的集合。类型可分为：

    • 原子类型
      每个数据都无法再分，如一个整数、浮点数、字符、指针等都是无法再分割的整体。
    • 结构类型
      结构类型由原子类型按照一定的规则构造而成，如一个银行账户一般包括姓名、地址、账号、余额等原子类型。
      数据结构是一种类型(一组值的集合)以及定义在此集合上的一组操作的总称。

      例如，对于bool类型来说，其值的范围为true、false，可用的操作有与、或、非等等。
      对于int类型，其值的范围为-2147483648～2147483647，可用的操作有：加、减、乘、除等等。

  • 抽象数据类型(Abstract Data Type,ADT)
    抽象数据类型通常是指由用户定义，用以表示应用问题的 数据模型，是将数据结构作为一个软件构件的实现。ADT只关心数据的逻辑结构和数据的运算，不关心数据的存储结构。
    抽象数据类型由基本的数据类型组成，并包括一组相关的操作。

    例如，C中struct，C++中的struct或class。

三要素

数据逻辑结构

指反映数据元素之间的逻辑关系的数据结构，其中的逻辑关系是指 数据元素之间的前后间关系，而与他们在计算机中的存储位置无关。逻辑结构包括：

  1. 集合
    数据结构中的元素之间除了“同属一个集合” 的相互关系外，别无其他关系。
  2. 线性结构
    数据结构中的元素存在一对一的相互关系。

    对于线性结构中的每一数据元素：
    除了第一个元素，所有元素都有唯一前驱;
    除了最后一个元素，所有元素都有唯一后继。

  3. 树形结构
    数据结构中的元素存在一对多的相互关系。
  4. 图结构
    数据结构中的元素存在多对多的相互关系。
    逻辑结构可分为线性结构和非线性结构，以上集合、树形结构、图结构均为非线性结构。

数据的存储结构(物理结构)

数据的逻辑结构在计算机存储空间中的存放形式称为数据的物理结构(也称为存储结构)。即如何用计算机表示数据元素的逻辑关系。一般来说，一种数据结构的逻辑结构根据需要可以表示成多种存储结构，常用的存储结构有：

  1. 顺序存储
    把逻辑上相邻的元素存储在物理位置上相邻的存储单元中，元素之间的逻辑关系由存储单元的邻接关系来体现。
  2. 链式存储
    逻辑上相邻的元素在物理位置上可以不相邻，借助指向元素存储位置的指针来表示元素之间的逻辑关系。
  3. 索引存储
    在存储元素信息的同时，还建立附加的索引表。索引表中的每一项称为索引项(索引表中的一条记录)，索引项的一般形式是： 其中关键码Key是能够唯一标识一个数据元素的数据项。如可以用学号唯一标识一个学生。
    索引又可分为：
    • 稠密索引(dense index)
      每一个结点在索引表中都有一个索引项。
      该索引项中的地址指向结点所在的物理位置。
    • 稀疏索引(sparse index)
      一组相邻的结点在索引表中只有一个索引项。
      该索引项中的地址指向一组相邻结点的起始位置。
  4. 哈希存储(散列存储)
    根据结点的关键码通过一个函数计算直接得到该结点的存储地址。
    1. 数据的存储结构会影响存储空间分配的方便程度。
      对于不同的问题、需求，要采用合适的存储结构。
    2. 数据的存储结构会影响对数据运算的速度。
    存储结构可分为顺序存储和非顺序存储，以上链式存储、索引存储和哈希存储均为非顺序存储。

数据的运算

施加在数据上的运算包括运算的定义和实现。
数据的运算是定义在逻辑结构上的，运算的实现要在存储结构上进行。

  • 一般有以下几种常用运算：
    1. 检索
      检索就是在数据结构里查找满足一定条件的节点。一般是给定一个某字段的值，找具有该字段值的结点。
    2. 插入
      往数据结构中增加新的结点。
    3. 删除
      把指定的结点从数据结构中去掉。
    4. 更新
      改变指定结点的一个或多个字段的值。
    5. 排序
      把结点按某种指定的顺序重新排列。例如递增或递减。
